Required models:

```python
class PreflightTaskSummary(BaseModel):
    task_type: str
    canonical_task_type: str
    dataset_dir: str
    ground_truth_path: str
    item_count: int
    selected_count: int
    warnings: list[str] = Field(default_factory=list)

class PreflightCostPreview(BaseModel):
    expected_request_count: int
    approximate_cost_usd: float | None = None
    pricing_source: str | None = None
    unavailable_reason: str | None = None

class PreflightReport(BaseModel):
    schema_version: str = PREFLIGHT_SCHEMA_VERSION
    run_id: str
    provider: str
    model: str
    prompt_mode: str
    selected_task_types: list[str]
    expected_request_count: int
    prompt_config: dict[str, Any]
    cost_preview: PreflightCostPreview
    output_dir: str
    manifest_path: str
    attempts_path: str
    tasks: list[PreflightTaskSummary]
    warnings: list[str] = Field(default_factory=list)
```

Required CLI flags:
- `--dataset-root` default `./captcha_data`
- `--types` nargs `+`, required
- `--prompts-file` default `./prompts_optimized.yaml`
- `--few-shot-config` default `None`
- `--prompt-prefix` default `None`
- `--prompt-suffix` default `None`
- `--pricing-file` default `None`; this must be a non-secret JSON/YAML metadata file if provided, not `secrets.yaml`
- `--output-root` default `./results/revision`
- `--run-id` required
- `--provider` required
- `--model` required
- `--prompt-mode` choices `auto`, `gt`, `opt`, default `auto`
- `--max-per-type` type `int`, default `None`
- `--max-attempts` type `int`, default `1`
- `--overwrite` action `store_true`
- `--resume` action `store_true`
- `--write-report` action `store_true`

Validation behavior:
- Normalize each requested task through `TASK_ALIASES`.
- Reject any canonical task not in `run_eval.SUPPORTED_TYPES`.
- Resolve dataset directory with `DATASET_DIR_ALIASES`; require `ground_truth.json`.
- Read `ground_truth.json` as text first and fail if it starts with `version https://git-lfs.github.com/spec/v1`.
- Parse ground truth JSON and require a non-empty list or dict.
- Validate that `--prompts-file` exists and YAML loads to a mapping when provided.
- Validate that `--few-shot-config` exists and YAML loads to a mapping when provided.
- Compute `prompt_config` with `prompts_file`, `prompts_file_sha256`, `few_shot_config`, `few_shot_config_sha256`, `prompt_prefix_sha256`, and `prompt_suffix_sha256` using `revision_artifacts.sha256_file()` and `sha256_text()`. Do not store raw inline prefix/suffix text in the report.
- Compute `selected_count = min(item_count, max_per_type)` when `max_per_type` is positive, else `item_count`.
- Compute `expected_request_count = sum(selected_count for tasks) * max_attempts`.
- Compute `cost_preview` from `--pricing-file` only when supplied. Support a non-secret mapping shaped like `pricing.<provider>.<model>.per_request_usd`; set `approximate_cost_usd = expected_request_count * per_request_usd` when present. If pricing metadata is absent, unreadable, lacks the provider/model, or lacks a per-request estimate, set `approximate_cost_usd = None` and an explicit `unavailable_reason` such as `"pricing metadata not provided"` or `"pricing metadata lacks per_request_usd for provider/model"`.
- Compute output paths using `revision_artifacts.revision_run_dir(output_root, run_id)`.
- Fail if the output directory exists and neither `--overwrite` nor `--resume` is set.
- Print the report as JSON to stdout.
- If `--write-report` is set, create the run directory and write `preflight_report.json`.

Do not import or call `run_eval.load_secrets`, `run_eval.make_provider`, `ModelProvider`, or any provider SDK class in this file.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI input -> dataset/prompt filesystem | Task names and paths are untrusted until validated. |
| Preflight -> provider-call boundary | Preflight must stop before secrets, provider clients, or paid requests. |
| Output path -> existing results | Preflight must prevent unintended overwrites before experiments start. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | Information Disclosure | `revision_preflight.py` | mitigate | Do not import or call secret loading/provider construction; tests monkeypatch provider creation to prove non-use. |
| T-01-04-02 | Denial of Wallet | Preflight CLI | mitigate | Report expected request count and fail before provider boundary. |
| T-01-04-03 | Tampering | Task aliases | mitigate | Canonical alias map and tests for `Connect_icon` -> `Connect_Icon`. |
| T-01-04-04 | Tampering | Dataset files | mitigate | Validate `ground_truth.json` existence, JSON parseability, non-empty contents, and Git LFS materialization. |
| T-01-04-05 | Tampering | Output directory | mitigate | Refuse existing `results/revision/<run_id>/` unless `--overwrite` or `--resume` is explicit. |
| T-01-04-06 | Repudiation | Preflight report | mitigate | Write `preflight_report.json` with schema version, selected tasks, counts, output paths, provider/model labels, and warnings. |
</threat_model>
