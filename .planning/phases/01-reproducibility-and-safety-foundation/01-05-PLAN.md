<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Revision-mode evaluator -> provider SDK | Manifest and output-directory checks must finish before provider construction. |
| Provider response -> attempt log | Per-attempt rows must be written before aggregate summaries. |
| Legacy evaluator behavior -> revision evidence | Known scoring/summary bugs can corrupt paper-facing outputs if untested. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | Information Disclosure | `run_eval.py` revision manifest | mitigate | Manifest uses provider/model labels and metadata only; no secret config fields. |
| T-01-05-02 | Denial of Wallet | `run_eval.py` revision mode | mitigate | Validate `write_attempts`, output directory policy, tasks, and manifest write before `make_provider()`. |
| T-01-05-03 | Tampering | `results/revision/<run_id>/` | mitigate | Refuse existing directories unless overwrite/resume flags are explicit. |
| T-01-05-04 | Repudiation | Attempt vs summary evidence | mitigate | Append `AttemptRecord` rows before aggregate summaries and derive revision summaries from `attempts.jsonl`. |
| T-01-05-05 | Tampering | Task/scoring regressions | mitigate | Add focused tests for `Path_Finder`, failure description, and summary CSV row counts. |
</threat_model>
