<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python import -> local filesystem | Importing modules must not read `secrets.yaml` or generated artifacts. |
| Explicit smoke CLI -> provider SDK | Paid provider calls can occur only after the user runs `revision_provider_smoke.py`. |
| Local secret file -> logs/artifacts | Secret-like values must pass through `revision_secrets.redact_mapping()` or `redact_text()` before output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Information Disclosure | `run_eval.py` import | mitigate | Remove import-time secret reads/prints and test sentinel/output leakage. |
| T-01-02-02 | Denial of Wallet | Provider smoke code | mitigate | Move provider smoke behavior to `revision_provider_smoke.py` under explicit `main()` only. |
| T-01-02-03 | Information Disclosure | `secrets.example.yaml` | mitigate | Commit only example tokens and no local credential values. |
| T-01-02-04 | Tampering | `.gitignore` | mitigate | Add local secret and `results/revision/` ignore entries to reduce accidental artifact churn. |
| T-01-02-05 | Repudiation | Tests/imports | mitigate | `tests/test_import_safety.py` captures stdout/stderr and fails on unsafe side-effect strings. |
</threat_model>
