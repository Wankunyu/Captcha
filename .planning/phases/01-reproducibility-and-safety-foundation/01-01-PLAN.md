name = "cognition-revision-experiments"

  "openai==2.6.1",
  "anthropic==0.72.0",

  "ruff==0.15.13",

[tool.ruff.lint]

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer machine -> dependency resolver | Dependency versions are resolved and written to `uv.lock`. |
| README -> researcher terminal | Commands may be copied into a local shell before later safety checks exist. |
| Local secret config -> documentation | Documentation references secret file names but must not contain credential values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | `README.md` | mitigate | Reference `secrets.example.yaml` and `secrets.yaml` by filename only; acceptance checks require no copied credential values. |
| T-01-01-02 | Denial of Wallet | README commands | mitigate | Document `revision_preflight.py` and local validation commands; do not document provider smoke as a default install step. |
| T-01-01-03 | Tampering | `uv.lock` | mitigate | Generate `uv.lock` from exact provider SDK pins and validate with `uv lock --check`. |
| T-01-01-04 | Repudiation | Dependency state | mitigate | Lock dependencies and configure pytest/ruff in `pyproject.toml` so runs can report the installed environment. |
| T-01-01-05 | Tampering | Generated revision outputs | transfer | This plan documents `results/revision/<run_id>/`; overwrite prevention is implemented in Plans 04 and 05. |
</threat_model>
