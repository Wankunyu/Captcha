`classify_rate()` must return:
- `"hard"` when `rate < cutoff - margin`
- `"borderline"` when `cutoff - margin <= rate <= cutoff + margin`
- `"broken"` when `rate > cutoff + margin`
- `None` when `rate is None`

Default `margin` is `0.0` to align Phase 2 comparison outputs with the submitted paper's 40% working threshold. A nonzero margin is an explicit analysis option only; broader threshold-sensitivity review belongs to Phase 3.

For every comparison row:
- `baseline_label = classify_rate(exp2_pass_at_1)`
- `adaptive_label = classify_rate(adaptive_success_rate)`
- `classification_change = f"{baseline_label}->{adaptive_label}"` when both labels exist, else `None`
- `cutoff_note = CUTOFF_NOTE`
- `structural_bottleneck_tags` is a semicolon-separated string from `STRUCTURAL_BOTTLENECK_TAGS`, with `"instruction sensitivity"` appended when `classification_change` is not `None` and labels differ
- `persistent_failure_note = "adaptive remained hard under binary-feedback explicit-memory budget"` only when `adaptive_label == "hard"`, `adaptive_observed_success is false`, and `scientific_wrong_count > 0`; otherwise `None`
- Infrastructure-only or protocol-only rows must never receive `persistent_failure_note`. If `adaptive_observed_success is false` and `scientific_wrong_count == 0` while `protocol_failure_count > 0` or `infrastructure_failure_count > 0`, set `persistent_failure_note = None` and preserve those failure counts for limitations/error-separation reporting.
- `confidence_interval_low` and `confidence_interval_high` remain `None` when the adaptive summary has no repeated-run CI, and `confidence_interval_not_applicable_reason` is `"single adaptive session; repeated-run CI deferred to Phase 3"`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Legacy CSVs -> comparison evidence | Existing Exp2/Exp3 outputs are normalized into main-body rows. |
| Adaptive summary -> comparison evidence | Adaptive outcomes and failure classes are interpreted for paper claims. |
| Output path args -> filesystem | CLI arguments choose where comparison CSV/JSON files are written. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Integrity | Baseline/adaptive merge | mitigate | Key rows by provider, model, task_type, and attempt_budget_k; missing fixed retry stays nullable. |
| T-02-04-02 | Repudiation | Failure interpretation | mitigate | Carry scientific, protocol, and infrastructure failure counts into comparison rows; emit persistent hard-family notes only when scientific failures are present. |
| T-02-04-03 | Information Disclosure | Comparison output | mitigate | Do not load secrets or write prompt/raw response/ground-truth fields into comparison outputs. |
| T-02-04-04 | Tampering | Output files | mitigate | Write only caller-specified CSV/JSON parent paths and validate rows through `AdaptiveComparisonRow`. |
| T-02-04-05 | Spoofing | Provider-native memory claims | mitigate | Include only explicit adaptive summary fields; no hidden provider session state appears in rows. |
| T-02-04-06 | Information Disclosure | Ground-truth leakage | mitigate | Comparison consumes aggregate rates and summaries only; tests assert no correct labels or coordinates are required. |
</threat_model>
