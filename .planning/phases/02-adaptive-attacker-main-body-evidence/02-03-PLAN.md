<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dataset task state -> provider prompt | Task prompt and images cross into paid model calls; ground truth must not. |
| Model raw/parsed output -> local memory | Only constrained policy notes may persist across fresh instances. |
| Provider/runtime errors -> scientific evidence | Runtime failures must remain separate from CAPTCHA robustness outcomes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | Reflection prompt builder | mitigate | Function accepts only current prompt, raw answer, parsed answer, and `Feedback: FAIL`; tests use sentinel ground truth and assert absence. |
| T-02-03-02 | Spoofing | Provider-native memory | mitigate | Each provider call is stateless and receives explicit local policy notes only; no provider session ids or hidden memory APIs are used. |
| T-02-03-03 | Tampering | Output paths | mitigate | Use `AdaptiveArtifactWriter`, which uses `revision_run_dir()` for `results/revision/<run_id>/`. |
| T-02-03-04 | Information Disclosure | Secrets | mitigate | Load local config only inside explicit run paths and never print or persist secret values; manifest stores provider/model labels only. |
| T-02-03-05 | Integrity | Failure classification | mitigate | Store `scientific_wrong`, `protocol_failure`, and `infrastructure_failure` separately and suppress reflection for non-scientific failures. |
| T-02-03-06 | Repudiation | Attempt stream | mitigate | Append `AdaptiveAttemptRecord` per solve attempt with policy state before/after, cumulative latency/cost, and stopping reason. |
</threat_model>
