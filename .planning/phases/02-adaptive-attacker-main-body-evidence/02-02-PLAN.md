<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> filesystem | User-provided paths and run ids select datasets, prompts, and output directories. |
| Prompt/few-shot files -> preflight report | File provenance enters durable report metadata as hashes only. |
| Request-count preview -> paid execution decision | Researchers use the report to decide whether to spend provider budget. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Information Disclosure | Preflight report | mitigate | Do not read `secrets.yaml`; read only non-secret `--pricing-file` metadata and hash prompt/few-shot inputs. |
| T-02-02-02 | Tampering | `--run-id` and `--output-root` | mitigate | Use `revision_run_dir()` and fail existing directories unless `--overwrite` or `--resume` is explicit. |
| T-02-02-03 | Information Disclosure | Adaptive semantics | mitigate | Report `feedback_mode=binary-pass-fail` and never include ground truth values in the preflight JSON. |
| T-02-02-04 | Spoofing | Provider memory | mitigate | Preflight records `memory_mode=explicit-policy-notes` and does not construct provider-native sessions. |
| T-02-02-05 | Repudiation | Request counts | mitigate | Report `solve_request_count`, `reflection_request_count_max`, and `expected_request_count_max` separately so reflection overhead cannot be hidden. |
| T-02-02-06 | Integrity | Scientific evidence | mitigate | Preflight is provider-free and cannot misclassify infrastructure failures as robustness evidence. |
</threat_model>
