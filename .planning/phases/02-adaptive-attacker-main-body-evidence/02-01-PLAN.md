<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local adaptive state -> durable artifacts | Explicit policy memory and attempts become paper evidence. |
| Run id -> filesystem path | User-provided run ids select output directories under `results/revision/<run_id>/`. |
| Attempt JSONL -> summary/comparison rows | Derived artifacts drive main-body evidence and must preserve failure classes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | `AdaptivePolicyState` | mitigate | Store only structured policy notes and reject banned tokens such as `ground_truth`, `correct_answer`, `raw_response`, and `transcript`. |
| T-02-01-02 | Tampering | `AdaptiveArtifactWriter` output paths | mitigate | Use `revision_run_dir()` for every run directory and test path traversal rejection. |
| T-02-01-03 | Information Disclosure | Artifact models | mitigate | Include provider/model labels and costs only; do not add fields for secret config or credential values. |
| T-02-01-04 | Spoofing | Adaptive memory semantics | mitigate | Persist local `policy_state_before` and `policy_state_after`; no provider-native hidden memory fields or session ids are modeled. |
| T-02-01-05 | Repudiation | Summary rows | mitigate | Count `scientific_wrong`, `protocol_failure`, and `infrastructure_failure` separately so infrastructure failures are not represented as CAPTCHA robustness. |
</threat_model>
