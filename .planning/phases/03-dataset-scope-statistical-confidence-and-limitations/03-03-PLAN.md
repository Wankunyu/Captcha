## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Legacy Exp2/Exp3 CSVs -> retry evidence | Existing aggregate result rows become prediction-calibration evidence. |
| Adaptive summary -> failure taxonomy | Phase 2 adaptive outcomes become scientific/protocol/infrastructure failure counts. |
| Calibration/failure logic -> paper claims | Derived rates and caveats influence manuscript robustness interpretation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | retry joins | mitigate | Join by provider, model, task_type, and attempt_budget_k; preserve task type as the primary comparison unit. |
| T-03-03-02 | Repudiation | prediction-error outputs | mitigate | Emit Exp2 pass@1, k, Bernoulli Success@k, observed outcomes, signed errors, absolute errors, and schema versions. |
| T-03-03-03 | Information Disclosure | retry/failure CLIs | mitigate | Read CSV/JSON artifacts only; do not read `secrets.yaml`, raw prompts, raw transcripts, or provider credential files. |
| T-03-03-04 | Tampering | failure interpretation | mitigate | Compute `scientific_rate` only from success and scientific-wrong counts; keep protocol/infrastructure failures caveated. |
| T-03-03-05 | Repudiation | aggregate-only legacy sources | mitigate | Label rows without failure classes as `aggregate_only_caveated` instead of using them for failure-taxonomy claims. |
| T-03-03-06 | Elevation of Privilege | live-service automation | accept | The plan adds offline analysis only and no browser automation, provider calls, or live CAPTCHA interaction. |
</threat_model>
