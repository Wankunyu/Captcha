## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local dataset files -> audit outputs | Ground-truth JSON and dataset directory names become paper-facing scope evidence. |
| Local curated manifest -> extended validation slice | User-curated extended rows become validation-slice metadata. |
| CLI run id/output paths -> filesystem | User-provided run IDs and output paths choose generated artifact locations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Information Disclosure | `dataset_scope_audit.py`, `extended_dataset_manifest.py` | mitigate | Do not read `secrets.yaml`; print only counts, paths, task names, schema versions, and non-secret manifest metadata. |
| T-03-01-02 | Tampering | output path handling | mitigate | Use `revision_run_dir(output_root, run_id)` for default outputs and create parents only under the resolved output root. |
| T-03-01-03 | Repudiation | dataset scope evidence | mitigate | Emit schema versions, run_id, sample counts, source paths, status fields, and exact reasons in CSV/JSON. |
| T-03-01-04 | Information Disclosure | extended dataset artifacts | mitigate | Require local manifest paths only; do not include raw prompts, provider responses, or credential values in contribution notes. |
| T-03-01-05 | Tampering | removed-task interpretation | mitigate | Hard-code the two removed task names and static-pipeline incompatibility reasons in tests so they cannot silently drift. |
| T-03-01-06 | Elevation of Privilege | live-service automation | accept | Phase 3 audit and manifest CLIs perform local file reads/writes only and introduce no browser automation or live CAPTCHA interaction. |
