```python
CUTOFF_NOTE = "40% working CAPTCHA threshold; operational reporting heuristic, not a universal CAPTCHA security boundary. 30%-50% is a revision-time caution band, not a new security tier."
def classify_threshold_label(rate: float | None, *, cutoff: float = 0.40, review_band_low: float = 0.30, review_band_high: float = 0.50, ci_crosses_cutoff: bool = False, trend_sensitive: bool = False) -> str | None

def write_threshold_outputs(rows: list[ThresholdSensitivityRow], output_csv: Path, output_json: Path) -> tuple[Path, Path]
```

Label rules:
- Return `None` when `rate is None`.
- Return `borderline/near-broken` when `review_band_low <= rate <= review_band_high`.
- Return `borderline/near-broken` when `ci_crosses_cutoff` is true.
- Return `borderline/near-broken` when `trend_sensitive` is true.
- Return `hard` when `rate < cutoff`.
- Return `broken` when `rate >= cutoff`.

For each task-type confidence row, set:
- `primary_experiment="exp2"` when an Exp2 row exists for the same provider/model/task_type; otherwise use the experiment on the row.
- `primary_rate` from the primary experiment row.
- `max_observed_rate` as the maximum pass rate for the same provider/model/task_type across Exp1, Exp2, Exp3, Exp4, adaptive summary rows, adaptive comparison rows, and any loaded extended validation-slice rows.
- `margin_to_cutoff = primary_rate - cutoff`.
- `in_30_50_review_band = review_band_low <= primary_rate <= review_band_high`.
- `ci_crosses_cutoff = ci_low is not None and ci_high is not None and ci_low <= cutoff <= ci_high`.
- `trend_sensitive = primary_rate < cutoff and max_observed_rate - primary_rate >= trend_delta`.
- `trend_sources` as an explicit semicolon-separated list of source names whose pass rate is at least `primary_rate + trend_delta`; allowed source names include `exp1`, `exp2`, `exp3`, `exp4`, `adaptive_summary`, `adaptive_comparison`, and `extended_validation_slice`.
- `cutoff_note = CUTOFF_NOTE`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Legacy result CSVs -> statistical evidence | Existing aggregate rows become confidence intervals and threshold labels. |
| CI and threshold logic -> paper claims | Derived labels can affect manuscript interpretation. |
| CLI run id/output paths -> filesystem | User-provided run IDs and paths choose generated statistical artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Tampering | result ingestion | mitigate | Load only explicit `exp1` through `exp4` result files and test that `results/error_analysis` is skipped. |
| T-03-02-02 | Repudiation | CI outputs | mitigate | Emit `ci_method`, `ci_confidence`, `n_attempts`, `n_success`, `source_path`, and schema version for every row. |
| T-03-02-03 | Information Disclosure | statistical CLI | mitigate | Read result CSVs only; do not read `secrets.yaml`, prompts, raw responses, or provider credentials. |
| T-03-02-04 | Tampering | cutoff interpretation | mitigate | Hard-code and test the operational-cutoff caveat and the 30%-50% review band. |
| T-03-02-05 | Denial of Service | malformed CSVs | mitigate | Fail with clear `ValueError` for unsupported schemas instead of silently producing partial confidence rows. |
| T-03-02-06 | Elevation of Privilege | live-service automation | accept | The plan adds offline analysis only and no browser automation or provider-call path. |
| T-03-02-07 | Tampering | output path handling | mitigate | Use `revision_run_dir(output_root, run_id)` for default outputs and test invalid/traversal run IDs. |
</threat_model>
