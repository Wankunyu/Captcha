<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| preflight matrix -> provider execution | A local JSON matrix authorizes paid provider calls. |
| provider responses -> revision artifacts | Untrusted model responses become attempts and summaries. |
| local secret config -> provider clients | Credentials are loaded only at approved execution boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.1-04-01 | Information Disclosure | static preflight reports | mitigate | Preflight must not read `secrets.yaml`; generated reports contain hashes, counts, paths, and cost preview objects only. |
| T-04.1-04-02 | Denial of Service | `collect-static` | mitigate | Require preflight request counts, `resume=true`, and explicit checkpoint approval before paid calls. |
| T-04.1-04-03 | Tampering | output directories | mitigate | Use `revision_artifacts.revision_run_dir()` through preflight/run paths and fail unless output dirs are resumable or unique. |
| T-04.1-04-04 | Repudiation | static summaries | mitigate | Write run manifests before provider construction, attempts before summaries, and summary rows with manifest/attempt references. |
| T-04.1-04-05 | Integrity | paper evidence | mitigate | Preserve scientific/protocol/infrastructure failure counts so provider failures are not represented as structural hardness evidence. |
</threat_model>
