## Decisions Made

- Kept the two new categories as offline static-image tasks only; no live service, browser automation, temporal hold, drag/slider, or production endpoint behavior was introduced.

- Used `reference_image` plus `option_images`/`options` for `Relation_Match`, matching the existing evaluator reference/options pattern.
- Exempted sidecar-only new task types from the legacy `captcha_data/` directory presence test because Phase 04.1 stores expanded data outside the original dataset tree.
