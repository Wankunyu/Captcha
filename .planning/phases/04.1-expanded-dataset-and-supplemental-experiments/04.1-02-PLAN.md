<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sidecar source files -> manifest validator | Local source files and manifests can be malformed, mislabeled, or non-authorized. |
| manifest -> evaluator_slice | Materialization copies source files into a run-evaluable directory. |
| repository source -> large dataset storage | Sidecar images can create repository/LFS and provenance risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.1-02-01 | Tampering | `load_phase041_manifest` | mitigate | Resolve `source_path` under `expanded_captcha_data/phase04_1/sources/`, require D-08 fields, and reject unsupported compatibility values. |
| T-04.1-02-02 | Information Disclosure | materializer | mitigate | Copy only referenced dataset files; do not read or write `secrets.yaml`; CLI summaries print row counts and paths only. |
| T-04.1-02-03 | Elevation of Privilege | category validation | mitigate | Reject rows requiring live service interaction, browser automation, temporal hold behavior, drag/slider execution, or production endpoint access. |
| T-04.1-02-04 | Tampering | original `captcha_data/` | mitigate | Materializer output root is `expanded_captcha_data/phase04_1/evaluator_slice`; tests assert original `captcha_data/` is not modified. |
| T-04.1-02-05 | Repudiation | sidecar provenance | mitigate | Require `source_id`, `source_path`, `metadata_alignment_notes`, `answer_format_normalization`, `limitation_notes`, and `adaptive_eligible` for every row. |
</threat_model>
