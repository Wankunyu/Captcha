# Phase 04.1 Dataset Definition Correction

## Corrected Definition

The Phase 04.1 expanded sidecar dataset is paper-eligible only when each evaluated row comes from one of these provenance classes:

1. Real CAPTCHA samples from a peer-reviewed paper dataset.
2. Real CAPTCHA samples from an open-source CAPTCHA dataset.
3. GPT Image generated CAPTCHA-style images explicitly modeled after the current Open CaptchaWorld dataset, with recorded generation provenance.

All rows must record `source_kind`, `source_citation`, `source_license`, and `source_provenance_notes` in addition to the existing manifest fields.

## Non-Eligible Prototype Rows

The current `Symbol_Count` and `Relation_Match` files under `expanded_captcha_data/phase04_1/sources/` were created as local scripted/PIL-style prototype fixtures during Phase 04.1. They are not real paper/open-source samples and are not GPT Image generated samples with recorded provenance.

Therefore, those rows must not be described as paper-eligible direct expanded sidecar evidence. Static, adaptive, and paper-output artifacts generated over the current 66-instance sidecar are provisional and must be rerun after replacing or regenerating these two categories under the corrected definition.

## Allowed Repair Paths

- Replace `Symbol_Count` and `Relation_Match` with real CAPTCHA samples from a cited paper or open-source CAPTCHA dataset.
- Or regenerate the two categories with GPT Image as Open CaptchaWorld-style images, store the prompts/generation metadata needed for provenance, and update the manifest fields accordingly.

## Downstream Rule

Phase 5 defense methodology and Phase 6 final paper-claim artifacts should not use the current scripted new-category rows as direct expanded-dataset evidence. They may use only corrected provenance-compliant sidecar outputs.
