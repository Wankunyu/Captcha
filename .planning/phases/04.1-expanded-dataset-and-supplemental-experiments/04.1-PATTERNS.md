## Scope Extraction

Phase 04.1 implies an additive revision layer over the existing Phase 1-4 contracts:

- A new Phase 04.1 artifact/schema module for expanded-dataset provenance, run matrix rows, static/adaptive summaries, paper table rows, and claim-boundary notes.
- A new CLI/orchestration module to validate the sidecar manifest, materialize an evaluator-compatible sidecar root, build preflight matrices, collect static/adaptive outputs, and generate paper outputs.
- Sidecar data artifacts under a new expanded dataset root, with `evaluator_slice/<TaskType>/ground_truth.json` layout for `run_eval`.
- Conditional task-contract edits in `run_eval.py`, `prompts_optimized.yaml`, and `visualize_results.py` if the two new static-compatible categories require new task types or answer shapes.
- Focused offline tests for schemas, manifest/materialization, static preflight/run contracts, adaptive eligibility/safety, task contracts, scoring, paper outputs, and secret-safe reports.
- Planning traceability updates because Phase 04.1 currently has `Requirements: TBD`.
- A sidecar data tracking decision for `expanded_captcha_data/**` because `.gitattributes` currently covers only `captcha_data/**`.

def load_ground_truth(type_dir: str) -> Dict[str, Any]:
    gt_path = os.path.join(type_dir, "ground_truth.json")

def revision_run_dir(output_root: str | Path, run_id: str) -> Path:
    if not _RUN_ID_RE.fullmatch(run_id):
        raise ValueError(
            "run_id must start with a letter or number and contain only letters, "
            "numbers, dot, underscore, or hyphen"

    root = Path(output_root).resolve()
    run_dir = (root / run_id).resolve()
    if not run_dir.is_relative_to(root):
